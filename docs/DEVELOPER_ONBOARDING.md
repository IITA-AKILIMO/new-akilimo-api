# Developer Onboarding

Everything you need to get the project running locally, understand the codebase, and start contributing.

---

## Prerequisites

| Tool | Version |
|------|---------|
| PHP | 8.3+ |
| Composer | 2.x |
| Node.js | 20+ |
| MariaDB | 11.2+ (or Docker) |
| Redis | 7.4+ (or Docker) |

## Local Setup

```bash
git clone https://github.com/IITA-AKILIMO/new-akilimo-api.git
cd new-akilimo-api

composer install
cp .env.example .env
php artisan key:generate
```

### Configure `.env`

Minimum required variables:

```env
# Database
DB_CONNECTION=mariadb
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=akilimo
DB_USERNAME=root
DB_PASSWORD=

# External compute service
AKILIMO_COMPUTE_BASE_URL=https://compute.example.com
AKILIMO_COMPUTE_ENDPOINT=/compute

# Auth
DEFAULT_ADMIN_PASSWORD=choose_a_strong_password

# Cache
CACHE_STORE=redis
CACHE_TTL=1h
REDIS_HOST=127.0.0.1
```

### Migrate and seed

```bash
php artisan migrate
php artisan db:seed          # seeds reference data + creates admin user (username: akilimo)
```

The seeder prints the admin API key and password to the console on first run. Store the key — it is shown only once.

### Start the dev server

```bash
composer dev         # API only, port 8600
composer dev:all     # API + queue worker + Vite
```

## Docker Setup (recommended)

```bash
docker compose up -d
```

This starts: API (PHP-FPM), MariaDB 11.2.3, Redis 7.4.1, Dozzle (log viewer at `:8080`).

Migrations run automatically on container start.

```
  Feature/                 HTTP-level tests (ComputeRecommendationTest, Auth/, etc.)

```

## Key Concepts

### Dual Authentication

`AuthenticateWithToken` middleware accepts either `Authorization: Bearer <token>` or `X-Api-Key: <key>`. Both mechanisms store only a SHA-256 hash in the DB — the raw value is never persisted. See [ARCHITECTURE.md](./ARCHITECTURE.md) for full details.

### Fertilizer Mapping

Before calling the compute service, `RecommendationService` loads available fertilizers from the DB for the request's country, then merges client-supplied prices/selection against them to build a flat key map. Adding a new fertilizer requires only a DB row — no code changes. See [ARCHITECTURE.md](./ARCHITECTURE.md) for the complete flow.

### DTOs

All data passed between layers uses Spatie LaravelData classes (`app/Data/`). `AkilimoComputeData::toArray()` overrides the default serialisation to merge the fertilizer flat-map directly at the root level of the Plumbr payload.

### Models

Models live in `app/Models/Base/` and are **auto-generated** by Reliese from the DB schema. Never hand-edit them. After any migration, run:

```bash
composer model:gen
```

### BaseJsonResource

All resource classes extend `BaseJsonResource` which provides `formatDate(mixed $value): ?string`. This returns ISO 8601 with the app timezone offset. Override it per-resource if a different format is needed.

## Adding a New Endpoint

The fastest path is the scaffold command:

```bash
php artisan make:api-scaffold ResourceName --model=ModelName --prefix=v1/resource-name
```

This generates: controller, repository, resource, resource collection, and inserts the route into `api.php`. Then:

1. Move the route to the correct group in `api.php` (public vs protected).
2. Fill in the resource's `toArray()` with the fields to expose.
3. Run `composer lint:fix-all`.
4. Write a feature test.

## Writing Tests

Tests use **Pest** with an SQLite in-memory database (configured in `phpunit.xml`).

Protected routes require authentication. Use the `actingAsApiUser()` helper from `TestCase`:

```php
beforeEach(fn () => $this->actingAsApiUser());

it('returns 200', function () {
    $this->getJson('/v1/some-endpoint')->assertOk();
});
```

To test with a specific ability scope, create the token manually:

```php
$user = User::factory()->create();
PersonalAccessToken::create([
    'tokenable_type' => User::class,
    'tokenable_id'   => $user->id,
    'name'           => 'test',
    'token'          => hash('sha256', $raw = bin2hex(random_bytes(32))),
    'abilities'      => ['read'],
]);
$this->withToken($raw);
```

To test API key authentication, flush the bearer header first:

```php
$this->flushHeaders()->withHeader('X-Api-Key', $rawKey);
```

## Admin Commands

```bash
# Create admin user and print credentials (first run only)
php artisan db:seed --class=AdminUserSeeder

# Rotate the admin API key
php artisan admin:regenerate-api-key

# Skip confirmation prompt
php artisan admin:regenerate-api-key --force

# Target a different username
php artisan admin:regenerate-api-key --username=otheradmin
```

## Branching & Commits

- Branch from `develop`; target `develop` for PRs.
- Use [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `refactor:`, `test:`, `docs:`, `chore:`.
- Keep PRs focused — one concern per PR.
- `main` is the production branch; only `develop` merges into it.
